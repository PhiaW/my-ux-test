# Recipe QuickCook — 設計摘要

> Phase 6 定稿（2026-06-04）。1–2 頁速覽本案的視覺語言與核心決策脈絡。
> 詳規格見 `design-system.md`，原則全文見 `design-principles.md`。

## 一句話定位

為「為孩子備餐的家庭煮食者」打造的純前端食譜查看 App——**打開冰箱、勾出有的食材，一眼看出今天能煮什麼**。不是食譜社群，是廚房裡的決策工具。

## 視覺語言

- **調性**：精緻生活感 × 留白簡潔。無彩暖灰為主體（60–90%），靠留白與襯線標題撐起質感，而非靠色彩或裝飾。
- **色彩雙重身分**：橄欖綠 `#688943`（primary）+ 暖黃 `#E0A23C`（secondary）同時是品牌色與**食譜燈號**——綠＝可做、黃＝差一樣/靠替代、紅＝食安警告。accent 只點睛，不鋪大底。
- **字體**：標題 Playfair Display（襯線、生活感）、內文 Noto Sans TC、計時數字 JetBrains Mono（等寬、讀秒穩定）。
- **形狀**：柔和圓角（btn 8 / card 16 / badge 膠囊），含蓄卡片陰影。

## 五條原則如何落地

1. **一眼可下手**——三模式以底部導覽（bottom nav）切換：依菜單挑／依食材選／備餐計畫；勾食材後卡片即時依「可做優先」排序、亮對照燈。情境／方法為兩個獨立的複選下拉（空選＝全部），選後可一鍵清除。
2. **濕手也能操作**——主要可點區 ≥44px（WCAG AA），計時數字 24px 大字；focus ring 一律保留。
3. **留白即精緻**——中性灰主導、accent 僅點睛。*例外*：極淺 `--primary-lt` 可作功能分組容器底色（如食材清單淺綠框），屬點睛而非彩色大底。
4. **備料安心**——食安提醒以紅框 `role="alert"` + 圖示強制顯示，對比達 AA（`error-strong`）；步驟內嵌倒數計時。
5. **這不是食譜社群**——無帳號、無探索流、無廣告/推薦；資料靜態開啟即用，原始影片只外跳 YouTube。

## 核心決策脈絡

- **燈號驅動的資訊架構**：每道菜的核心食材逐項判 HAVE/SUBBED/MISSING，聚合成單一燈號——把「能不能煮」這個決策前置到列表層，符合「一眼可下手」。
- **採買清單而非購物車**：勾選缺料 → 一鍵複製文字清單，不做帳號/結帳。符合「不是食譜社群」。
- **Figma ↔ Code 雙向**：本案走 ds-template Phase 1–6，元件先在 Figma 綁 token/Text Style，再回灌 React CSS；所有值走 CSS 變數、結構 flex-first 可倒回 Figma Auto Layout。
- **a11y 為硬約束**：對比全 AA、觸控 ≥44、focus ring 不可移除——這些在 Phase 5 稽核時反過來修正了 Principle 2 的初稿數值。

## 已知取捨與後續

- 食材勾選刻意不存 localStorage（避免「常備清單」變成隱性帳號）；未來若做需明確設計。
- 食材破百再加搜尋框（濕手情境僅輔助）。
- 功能 backlog（非 ds-template 流程）：React Router、採買頁/備料指引頁、來源 icon 判斷。
