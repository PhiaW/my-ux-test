Button

按鈕是使用者與任性選擇器互動最主要的元件。共有 8 個變體，各自對應不同的操作語意與視覺重量，確保使用者在每個畫面都能直觀判斷操作的優先順序。

Overview

何時使用按鈕
按鈕用於觸發即時的、確定性的操作，例如確認選擇、拒絕結果、返回上一步。每個畫面的主要操作應使用最高重量的按鈕（btn-main），次要操作降一層。
何時不用按鈕
用於導覽跳轉（如外開 Google Maps）時應改用帶有 ↗ 箭頭的 .maps-search-card 連結卡片。純文字返回操作請使用 .back-link 而非按鈕。

Anatomy

以 btn-main 為基準解析按鈕的組成元素，其他變體共用相同結構但調整外觀屬性。

Label
按鈕文字，說明操作內容。字型 Noto Sans TC，font-size 依 variant 為 --fs-body (18px) 或 --fs-label (16px)。文字應清楚描述動作，避免「確認」、「OK」等模糊詞。
3
Icon（選用）
以 emoji 形式出現，放置於 label 前方。用於強化語意（📍 找附近、🙅 拒絕、🎁 驚喜）。圖示應與文字意義一致，不可單獨使用。

Variants

8 個按鈕變體，依視覺重量由高至低排列。每個畫面最多只能有一個 btn-main。

Primary .btn-main
最高視覺重量。每個畫面唯一的主要操作，例如「確認選擇」、「重新抽選」。背景色 --primary，文字色 --on-primary，圓角 --rs。font-size --fs-body (18px)。
Ghost .btn-ghost
次要操作。與 btn-main 並排時承擔「拒絕/返回」的語意。透明背景 + border，hover 時 text/border 變 --primary。font-size --fs-label (16px)。
Dashed .btn-cantfind
輔助操作。以 dashed border 降低視覺重量，讓使用者知道此操作「可選但不是重點」。hover 時 border 變 solid + primary 色。
Restart .btn-restart
專用於 .final-card（primary 底色背景）內。白色半透明 pill 形狀，讓按鈕在 primary 背景上仍能被識別，但不搶過最終結果。
Calm .btn-calm
彩蛋畫面「冷靜了」操作。Pill 形狀使用 --bg 背景 + --border 邊框，在 exhausted-card 的 Surface 背景上呈現低強度感。
Blind .btn-blind
盲選挑戰 CTA。Primary 背景 + pill 形狀，與 btn-calm 並排時擔任視覺焦點，引導使用者嘗試驚喜玩法。hover 有 translateY(-2px) 效果強調互動感。
Icon Round .theme-toggle
38×38px 圓形 icon-only 按鈕，固定出現於右上角。Surface 背景 + border，hover 時背景變 primary-lt。此 variant 不放文字 label。

Text Link .back-link
無背景、無邊框的純文字操作，視覺重量最低。專門用於「返回上一步」的導覽行為，不應用於確認型操作。hover 時文字色變 primary。

Disabled 實作規則
套用 opacity: .4 + cursor: not-allowed + pointer-events: none。目前 App 中不主動使用 disabled 狀態，但在 loading 或等待動畫期間可加上以防重複觸發。

Usage Guidelines

操作層級
✓Do
每個畫面只有一個 btn-main，次要操作用 ghost 或 cantfind，層級清楚。
✗Don't
同一畫面出現兩個 btn-main，使用者無法判斷哪個才是主要操作。
按鈕文字
✓Do
文字包含動作結果，使用者明確知道點下去會發生什麼。
✗Don't
「確認」、「OK」、「Submit」等模糊詞無法傳達操作語意。
形狀語意
✓Do
主要操作用 rounded（--rs），情境型輔助操作（彩蛋、重啟）用 pill，形狀輔助語意區分。
✗Don't
把主要確認操作套用 pill 樣式，破壞形狀語意一致性。
在 Primary 底色上使用按鈕
✓Do
在 final-card（primary 背景）內使用 btn-restart，白色半透明確保對比度。
✗Don't
在 primary 背景上使用 btn-main，顏色幾乎相同，對比度不足。
